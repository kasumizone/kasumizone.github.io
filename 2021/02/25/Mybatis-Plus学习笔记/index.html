1.1 主键生成策略

雪花算法：

Twitter的snowflake算法 几乎保证全球唯一

snowflake是Twitter开源的分布式ID生成算法，结果是一个long型的ID。其核心思想是：使用41bit作为毫秒数，10bit作为机器的ID（5个bit是数据中心，5个bit的机器ID），12bit作为毫秒内的流水号（意味着每个节点在每毫秒可以产生 4096 个 ID），最后还有一个符号位，永远是0。具体实现的代码可以参看https://github.com/twitter/snowflake。

不勾选自增，也不配置TableId 的话，默认是唯一ID

4. 逻辑删除(deleted)

物理删除：从数据库中直接删除

逻辑删除：在数据库中仍然存在，通过改变一个值让他失效deleted=0 ==> deleted=1

管理员可以查看被删除的记录，类似于回收站

  1. 在数据库中添加以下字段：将其默认值设置为0，表示未被删除

  1. 在实体类中增加对于的属性与注解
@TableLogic     //逻辑删除
private Integer deleted;
  1. 在yaml（或properties）配置文件中进行配置
mybatis-plus:
  global-config:
    db-config:
      logic-delete-value: 1				#删除了的为1
      logic-not-delete-value: 0			#没被删除的为0
  1. 测试：

四、乐观锁（version）(旧版本)

乐观锁 : 故名思意十分乐观，它总是认为不会出现问题，无论干什么不去上锁！如果出现了问题， 再次更新值测试

悲观锁：故名思意十分悲观，它总是认为总是出现问题，无论干什么都会上锁！再去操作！

乐观锁的实现方式：

* 取出记录时，获取当前version属性的值
* 更新时，检查version属性的值是否符合要求，如果否，则不执行更新操作
* 执行更新时，对version值进行修改

测试：

  1. 给数据库中加version字段，默认值为1

  1. 实体类加上对应字段并标注@Version注解
@Version   //乐观锁version注解
private Integer version;
  1. 注册组件(config包)
@Configuration //配置类
public class MybatisPlusConfig {
//    注册乐观锁插件
    @Bean
    public OptimisticLockerInterceptor optimisticLockerInterceptor() {
        return new OptimisticLockerInterceptor();
    }

}

总结

多线程操作同一个对象的情况下，在第一个线程还没有更新的情况下，插队线程先修改并完成更新，如果没有乐观锁的话，那么第一个线程会覆盖掉第二个插队线程修改的值，有了乐观锁，第一个线程就不会覆盖插队线程修改的值，因为插队线程修改时，已经将乐观锁的version值+1了，第一个线程在进行修改是判断version的值与条件不符，所以将不会执行修改操作。

需要mybatis-plus版本 >= 3.0.7 param 参数名要么叫ew,要么加上注解@Param(Constants.WRAPPER) 使用${ew.customSqlSegment} 不支持 Wrapper 内的entity生成where语句

@Select("select * from mysql_data ${ew.customSqlSegment}")

2.2 用XML

//按条件分页查询课程列表
@Override
public IPage<CoursePublishVo> getCourseDisplayListService(Integer current, Integer limit, CourseQuery courseQuery) {

    Page<CoursePublishVo> page = new Page<>(current,limit);

    QueryWrapper<CourseQuery> wrapper = new QueryWrapper<>();
    String title = courseQuery.getTitle();
    String teacherName = courseQuery.getTeacherName();
    String subjectParentName = courseQuery.getSubjectParentName();
    String subjectName = courseQuery.getSubjectName();
    if (!StringUtils.isEmpty(title)){
        wrapper.like("ec.title",title);
    }
    if (!StringUtils.isEmpty(teacherName)){
        wrapper.eq("ec.name",teacherName);
    }
    if (!StringUtils.isEmpty(subjectParentName)){
        wrapper.eq("es1.title",subjectParentName);
    }
    if (!StringUtils.isEmpty(subjectName)){
        wrapper.eq("es2.title",subjectName);
    }
    IPage<CoursePublishVo> coursePage = baseMapper.getCourseDisplayList(page, wrapper, current, limit, courseQuery);
    return coursePage;
}
