一、Shiro核心组件

用户，角色，权限

给角色赋予权限，给用户赋予角色

  1. UsernamePasswordToken，shiro用来封装用户登录信息，使用用户的登录信息来创建令牌 Token 。
  2. SecurityManager，Shiro核心部分，负责安全认证和授权。
  3. Subject，Shiro的一个抽象概念，包含了用户信息。
  4. realm，开发者自定义的模块，根据项目的需求，验证和授权的逻辑全部写在 Realm 中。
  5. AuthenticationInfo，用户的角色信息集合，认证时使用。
  6. AuthorizationInfo，角色的权限信息集合，授权时使用
  7. DefaultWebSecurityManager，安全管理器，开发者自定义的Realm需要注入到 DefaultWebSecurityManager 进行管理才能生效。
  8. ShiroFilterFactoryBean，过滤器工厂，Shiro的基本运行机制是开发者定制规则，Shiro去执行，具体的执行操作就是由 ShiroFilterFactoryBean 创建的一个个Filter对象来完成的。

2. 自定义Shiro过滤器

在realm包下创建AccountRealm类

package lover.alice.realm;

import lover.alice.entity.Account;
import lover.alice.service.AccountService;
import org.apache.shiro.authc.*;
import org.apache.shiro.authz.AuthorizationInfo;
import org.apache.shiro.realm.AuthorizingRealm;
import org.apache.shiro.subject.PrincipalCollection;
import org.springframework.beans.factory.annotation.Autowired;

public class AccountRealm extends AuthorizingRealm {		//继承这个类，并且实现其中的两个方法

    @Autowired
    AccountService accountService;
    /**
     * 授权
     */
    @Override
    protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principalCollection) 	{
        return null;
    }
    
    /**
     * 认证
     * 登录验证
     */
    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken authenticationToken) throws AuthenticationException {

        //拿到一个token，封装拿到的username和password，
        //这个token已经传进来了就是authenticationToken，给他强转成UsernamePasswordToken。
        UsernamePasswordToken token = (UsernamePasswordToken) authenticationToken;
        //拿到token中传来的username，在数据库中查出对应的Account
        Account account = accountService.getAccountByName(token.getUsername());
        if (account!=null){       //判断用户名存在
            /**
             * 进行密码验证，用这个方法 SimpleAuthenticationInfo() 直接进行密码验证，直接返回这个对象
             * 参数一：通过用户名查出来的对象
             * 参数二：正确的密码
             * 参数三：getName()
             */
            return new SimpleAuthenticationInfo(account,account.getPassword(),getName());
        }
        return null;
    }
}

3. 配置类config

进行配置才能生效

package lover.alice.config;

import lover.alice.realm.AccountRealm;
import org.apache.shiro.spring.web.ShiroFilterFactoryBean;
import org.apache.shiro.web.mgt.DefaultWebSecurityManager;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class ShiroConfig {

    /**
     * 创建一个AccountRealm对象并且通过 @Bean 把他放到IOC容器中
     * 因为开发者自定义的Realm需要注入到 DefaultWebSecurityManager 进行管理才能生效。
     * 所以下一步
     */
    @Bean
    public AccountRealm accountRealm(){
        return new AccountRealm();
    }

    /**
     * @param accountRealm  上一步注册到IOC中的对象，现在通过 @Qualifier 取出来并且注入到DefaultWebSecurityManager中
     * @Qualifier 通过对象的名字，从容器中取出需要的对象，名字默认是方法名
     * 同样要把它注入到IOC容器中，供最后干事的 ShiroFilterFactoryBean 使用
     * 所以下一步
     */
    @Bean
    public DefaultWebSecurityManager securityManager(@Qualifier("accountRealm") AccountRealm accountRealm){
        DefaultWebSecurityManager manager = new DefaultWebSecurityManager();
        manager.setRealm(accountRealm);
        return manager;
    }

    /**
     * @Qualifier 同样通过它拿到 securityManager ，
     * 注入进 ShiroFilterFactoryBean 即可
     */
    @Bean
    public ShiroFilterFactoryBean shiroFilterFactoryBean(@Qualifier("securityManager") DefaultWebSecurityManager securityManager){
        ShiroFilterFactoryBean factoryBean = new ShiroFilterFactoryBean();
        factoryBean.setSecurityManager(securityManager);
        return factoryBean;
    }


}

4. 认证和授权

认证过滤器：

anon：无需认证；

anthc：必须认证；

anthcbasic：需要通过 HTTPBasic 认证；

user：不一定需要认证，只要曾经被 Shiro 记住即可（记住我）；

授权过滤器

perms：必须拥有某个权限才能访问；

roles：必须拥有某个角色才能访问；

port：请求的端口必须是指定值才可以访问；

rest：请求必须基于 RESTful：POST，GET，PUT，DELETE；

ssl：必须是安全的URL请求，协议HTTPS。

5. 配置认证和授权

创建 3 个页面，main.html、manage.html、administrator.html

访问权限如下：

1、必须登录才能访问 main.html

2、当前用户必须拥有 manage 授权才能访问 manage.html

3、当前用户必须拥有 administrator 角色才能访问 administrator.html

5.1 配置类中配置

在config配置类中，对访问三个页面的 请求 进行限制

操作的是**ShiroFilterFactoryBean方法中的factoryBean**对象

@Bean
public ShiroFilterFactoryBean shiroFilterFactoryBean(@Qualifier("securityManager") DefaultWebSecurityManager securityManager){
    ShiroFilterFactoryBean factoryBean = new ShiroFilterFactoryBean();
    factoryBean.setSecurityManager(securityManager);

    //权限设置，限制请求的访问权限
    Map<String,String> map = new HashMap<>();               //规则存放在HashMap中
    map.put("/main","authc");                               
    //必须登录才能访问 main.html("/main"请求)
    map.put("/manager","perms[manager]");                   
    //当前用户必须拥有 manage 授权才能访问 manage.html("/manager"请求)
    map.put("/administrator","roles[administrator]");       
    //当前用户必须拥有 administrator 角色才能访问 administrator.html("/administrator"请求)
    factoryBean.setFilterChainDefinitionMap(map);           //把定义好的规则放进去
    
    //设置登录页面
    factoryBean.setLoginUrl("/login");
    
    return factoryBean;
}

这样就对访问这三个页面的请求进行了限制

这样再访问的话，就会默认自动跳转到 login.jsp 页面，但是我们没有这样的页面，

需要定制一个login页面并且把没有权限访问的请求自动跳转到我们定制的login页面

要写一个login页面，Controller要有一个”/login”请求到这个页面，

然后配置

factoryBean.setLoginUrl("/login");

5.2 处理login登录验证的Controller

@GetMapping("{url}")                            //转到页面的处理
public String tourl(@PathVariable String url){
    return url;
}

@PostMapping("/login")
public String login(String username, String password, Model model){
    //使用 Shiro 进行处理
    //拿到Subject： Shiro的一个抽象概念，包含了用户信息。
    Subject subject = SecurityUtils.getSubject();
    //把用户输入的信息传到token中封装
    UsernamePasswordToken token = new UsernamePasswordToken(username,password);
    /**
     * 执行了下面这一步，就会把登录请求交给Shiro去做，
     * 进入到 AccountRealm 中，对拿到的token，对比数据库，执行登录验证的操作，
     * 在 AccountRealm 中执行时，
     * 用户名错误，也就是根据用户名查不到数据库中对应的对象，会返回UnknownAccountException异常
     * 密码错误，返回IncorrectCredentialsException异常
     * 我们可以在这里捕获异常，并返回给登录页面错误信息提示
     */
    try {
        subject.login(token);
        return "index";
    } catch (UnknownAccountException e) {
        e.printStackTrace();
        model.addAttribute("msg","用户名不存在！");
        return "login";

    } catch (IncorrectCredentialsException e){
        model.addAttribute("msg","密码错误！");
        return "login";
    }

}

5.3 授权

使用Set集合，防止重复。

	/**
     * 授权
     */
@Override
protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principalCollection) {

    //获取当前登录的用户信息
    Subject subject = SecurityUtils.getSubject();
    Account account = (Account) subject.getPrincipal();                 
    //把认证时传进去的 account 对象拿出来，这是登陆成功才会传进去的对象

    //设置角色
    Set<String> roles = new HashSet<>();
    roles.add(account.getRole());                  //获取数据库中存放的用户的角色信息，加到集合中
    SimpleAuthorizationInfo info = new SimpleAuthorizationInfo(roles);  
    //将用户的角色信息上传到SimpleAuthorizationInfo中

    //设置权限
    info.addStringPermission(account.getPerms());   //获取数据库中的权限信息，上传到SimpleAuthorizationInfo中

    //上传
    return info;
}

2. Realm类

package lover.alice.realm;

import lover.alice.entity.Account;
import lover.alice.service.AccountService;
import org.apache.shiro.SecurityUtils;
import org.apache.shiro.authc.*;
import org.apache.shiro.authz.AuthorizationInfo;
import org.apache.shiro.authz.SimpleAuthorizationInfo;
import org.apache.shiro.realm.AuthorizingRealm;
import org.apache.shiro.subject.PrincipalCollection;
import org.apache.shiro.subject.Subject;
import org.springframework.beans.factory.annotation.Autowired;

import java.util.HashSet;
import java.util.Set;

public class AccountRealm extends AuthorizingRealm {

    @Autowired
    AccountService accountService;
    /**
     * 授权
     */
    @Override
    protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principalCollection) {

        //获取当前登录的用户信息
        Subject subject = SecurityUtils.getSubject();
        Account account = (Account) subject.getPrincipal();                 //把认证时传进去的 account 对象拿出来，这是登陆成功才会传进去的对象

        //设置角色
        Set<String> roles = new HashSet<>();
        roles.add(account.getRole());                  //获取数据库中存放的用户的角色信息，加到集合中
        SimpleAuthorizationInfo info = new SimpleAuthorizationInfo(roles);  //将用户的角色信息上传到SimpleAuthorizationInfo中

        //设置权限
        info.addStringPermission(account.getPerms());   //获取数据库中的权限信息，上传到SimpleAuthorizationInfo中

        //上传
        return info;
    }

    /**
     * 认证
     * 登录验证
     */
    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken authenticationToken) throws AuthenticationException {

        //拿到一个token，封装拿到的username和password，
        //这个token已经传进来了就是authenticationToken，给他强转成UsernamePasswordToken。
        UsernamePasswordToken token = (UsernamePasswordToken) authenticationToken;
        //拿到token中传来的username，在数据库中查出对应的Account
        Account account = accountService.getAccountByName(token.getUsername());
        if (account!=null){       //判断用户名存在
            /**
             * 进行密码验证，用这个方法 SimpleAuthenticationInfo() 直接进行密码验证，直接返回这个对象
             * 参数一：被验证的对象
             * 参数二：正确的密码
             * 参数三：getName()
             */
            return new SimpleAuthenticationInfo(account,account.getPassword(),getName());
        }
        return null;
    }
}

4. Controller层

package lover.alice.controller;

import lover.alice.entity.Account;
import org.apache.shiro.SecurityUtils;
import org.apache.shiro.authc.AuthenticationException;
import org.apache.shiro.authc.IncorrectCredentialsException;
import org.apache.shiro.authc.UnknownAccountException;
import org.apache.shiro.authc.UsernamePasswordToken;
import org.apache.shiro.subject.Subject;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.*;

@Controller
public class ShiroController {
    @GetMapping("{url}")                            //转到页面的处理
    public String tourl(@PathVariable String url){
        return url;
    }

    @PostMapping("/login")
    public String login(String username, String password, Model model){
        //使用 Shiro 进行处理
        //拿到Subject： Shiro的一个抽象概念，包含了用户信息。
        Subject subject = SecurityUtils.getSubject();
        //把用户输入的信息传到token中封装
        UsernamePasswordToken token = new UsernamePasswordToken(username,password);
        /**
         * 执行了下面这一步，就会把登录请求交给Shiro去做，
         * 进入到 AccountRealm 中，对拿到的token，对比数据库，执行登录验证的操作，
         * 在 AccountRealm 中执行时，
         * 用户名错误，也就是根据用户名查不到数据库中对应的对象，会返回UnknownAccountException异常
         * 密码错误，返回IncorrectCredentialsException异常
         * 我们可以在这里捕获异常，并返回给登录页面错误信息提示
         */
        try {
            subject.login(token);

            //拿到登录成功的用户的信息，存到Session中
            Account account = (Account) subject.getPrincipal();
            subject.getSession().setAttribute("account",account);

            return "index";
        } catch (UnknownAccountException e) {
            e.printStackTrace();
            model.addAttribute("msg","用户名不存在！");
            return "login";

        } catch (IncorrectCredentialsException e){
            model.addAttribute("msg","密码错误！");
            return "login";
        }

    }

    @GetMapping("/unauth")
    @ResponseBody
    public String unauth(){
        return "未授权，无法访问！";
    }


    @GetMapping("/logout")
    public String logout(){
        Subject subject = SecurityUtils.getSubject();
//        subject.getSession().removeAttribute("account");
        subject.logout();  //直接调这个方法 ，直接销毁Session
        return "login";
    }

}
