RestTemplate组件

ConsumerController中进行调用provider实例中的 index 获取端口号的方法

@ResponseBody
@RequestMapping("/index")
public String index(){
    //1.拿到provider的某一个实例
    List<ServiceInstance> provider = this.discoveryClient.getInstances("provider");
    int index = ThreadLocalRandom.current().nextInt(provider.size());   
    //随机数，随即调用几个provider中的某一个//真实项目不这么写，测试
    ServiceInstance serviceInstance = provider.get(index);              //拿到一个provider实例

    String url = serviceInstance.getUri()+"/index";                     //拼出完整调用路径
    //调用
    String res = restTemplate.getForObject(url, String.class);//参数一：调用的URL，参数二：调用的方法的返回值类型
    return "调用了端口号为"+serviceInstance.getPort()+"的服务，得到的结果为"+res;
}

调用就会随机调用provider中的一个，刷新端口号会变

二、Ribbon

引入Spring Cloud，自动引入Ribbon

对 RestTemplate 的整合

1. RestTemplate配置类上加注解

加 ==@LoadBalance== 注解

@Configuration
public class ConsumerConfig {

    @Bean
    @LoadBalanced
    public RestTemplate restTemplate(){
        return new RestTemplate();
    }
}

2. ConsumerController 中调用provider中的方法

@ResponseBody
@RequestMapping("/index")
public String index(){
    return restTemplate.getForObject("http://provider/index",String.class);	//只需要这样写URL即可
}

直接调用RestTemplate的方法，==URL：http://提供者实例名/提供者的方法== 即可完成调用

会发现调用的provider实例是按顺序循环调用的

4. 权重算法

基于nacos中对每个实例设置的权重进行选择，权重越大的实例被选中的概率越高

写配置类：类名 NacosWeightedRule

import com.alibaba.cloud.nacos.NacosDiscoveryProperties;
import com.alibaba.cloud.nacos.ribbon.NacosServer;
import com.alibaba.nacos.api.exception.NacosException;
import com.alibaba.nacos.api.naming.NamingService;
import com.alibaba.nacos.api.naming.pojo.Instance;
import com.netflix.client.config.IClientConfig;
import com.netflix.loadbalancer.AbstractLoadBalancerRule;
import com.netflix.loadbalancer.BaseLoadBalancer;
import com.netflix.loadbalancer.ILoadBalancer;
import com.netflix.loadbalancer.Server;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;

@Slf4j
public class NacosWeightedRule extends AbstractLoadBalancerRule {

    @Autowired
    private NacosDiscoveryProperties nacosDiscoveryProperties;

    @Override
    public void initWithNiwsConfig(IClientConfig iClientConfig) {
        //读取配置文件
    }

    @Override
    public Server choose(Object o) {
        ILoadBalancer loadBalancer = this.getLoadBalancer();
        BaseLoadBalancer baseLoadBalancer = (BaseLoadBalancer) loadBalancer;
        //获取要请求的微服务名称
        String name = baseLoadBalancer.getName();
        //获取服务发现的相关API
        NamingService namingService = nacosDiscoveryProperties.namingServiceInstance();
        try {
            Instance instance = namingService.selectOneHealthyInstance(name);
            log.info("选择的实例是port={},instance={}",instance.getPort(),instance);
            return new NacosServer(instance);
        } catch (NacosException e) {
            e.printStackTrace();
            return null;
        }
    }
}

三、Sentinel 服务限流降级

1. 降级，限流，熔断

降级：访问量大的出现，限制一些功能的使用，只提供部分功能

        比如之前能提供5个功能，限制访问的太多，只给你提供三个功能，来减轻压力

限流：比如规定10个访问量，第11个就进不来，给你抛异常

熔断：比如要调用别人，对方出现故障，则熔断不允许访问

降级和限流是应对自身系统的故障，熔断是应对外部系统的故障

3. 流控模式

  • 流控模式：

    • 直接
    • 关联：对/index设置一个关联资源/list，list访问量超过阈值，对/index进行限流

    • 链路：直接和关联限流只能对Controller层入口进行限流，链路限流可针对其他入口进行限流

      • 高版本1.7以上Sentinel需要进行配置：默认对其他入口收敛，需要配置开放他们，进行限流

        • pom文件中加入依赖：

          <dependency>
    <groupId>com.alibaba.csp</groupId>
    <artifactId>sentinel-core</artifactId>
    <version>1.7.1</version>
</dependency>

<dependency>
    <groupId>com.alibaba.csp</groupId>
    <artifactId>sentinel-web-servlet</artifactId>
    <version>1.7.1</version>
</dependency>

        • 加入以下yaml配置

          spring:
  cloud:
    sentinel:
      filter:
        enabled: false

        • 定义Filter配置类：

          package lover.alice.config;


import com.alibaba.csp.sentinel.adapter.servlet.CommonFilter;
import org.springframework.boot.web.servlet.FilterRegistrationBean;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class FilterConfig {
    @Bean
    public FilterRegistrationBean registrationBean(){
        FilterRegistrationBean registrationBean = new FilterRegistrationBean();
        registrationBean.setFilter(new CommonFilter());
        registrationBean.addUrlPatterns("/*");
        registrationBean.addInitParameter(CommonFilter.WEB_CONTEXT_UNIFY,"false");
        registrationBean.setName("sentinelFilter");
        return registrationBean;
    }
}

4. 流控效果

  • 快速失败：流量超标->抛异常

  • Warm Up

    程序刚启动，需要预热过程，刚启动到达到最佳状态需要预热时间

  • 排队等待

    请求调用失败，不会立即抛异常，会等待下一次调用，下次调用可以则调通，调不通则抛异常

6. 热点规则

热点规则是流控规则的更细粒度操作，可以具体到对某个热点参数的限流，设置限流之后，如果带着限流参数的请求量超过阈值，则进行限流，时间为统计窗口时长。

必须要添加 **@SentinelResource**，即对资源进行流控。

@ResponseBody
@GetMapping("/hot")
@SentinelResource("hot_Sentinel")
public String hotMethod(
        @RequestParam(value = "num1",required = false) Integer num1,
        @RequestParam(value = "num2",required = false) Integer num2){
    return num1+"-"+num2;
}

7. 授权规则

给指定的资源设置流控应用（追加参数），可以对流控应用进行访问权限的设置，具体就是添加白名单和黑名单。

如何给请求指定流控应用，通过实现 RequestOriginParser 接口来完成，代码如下所示。

配置下面的配置后，访问任何接口都需要带参数 **“name”**才能进行访问

package lover.alice.config;

import com.alibaba.csp.sentinel.adapter.servlet.callback.RequestOriginParser;
import org.springframework.util.StringUtils;

import javax.servlet.http.HttpServletRequest;

public class RequestOriginParserDefinition implements RequestOriginParser {
    @Override
    public String parseOrigin(HttpServletRequest httpServletRequest) {
        String name = httpServletRequest.getParameter("name");
        if(StringUtils.isEmpty(name)){
            throw new RuntimeException("name is null");
        }
        return name;
    }
}

要让 RequestOriginParserDefinition 生效，需要在配置类中进行配置。

@Configuration
public class SentinelConfiguration {

    @PostConstruct
    public void init(){
        WebCallbackManager.setRequestOriginParser(new RequestOriginParserDefinition());
    }
}
